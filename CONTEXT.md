# UWB Light Trail — 领域术语表

## 核心概念

| 术语 | 定义 |
|------|------|
| **UWB 定位** | 通过 4 个固定基站(A0-A3)的测距数据，使用最小二乘法解算标签(Tag)的三维坐标 |
| **基站 (Anchor)** | 位置已知的 UWB 收发节点，共有 4 个 (A0~A3)，每个有 (X,Y,Z) 坐标 |
| **标签 (Tag)** | 附着在被追踪物体上的 UWB 移动节点，实时解算其三维位置 |
| **目标点 (Target)** | 标签期望到达的导航目标位置 (X,Y,Z)，2D 中以红色星形标记，3D 中以红色球体 + Sprite 文字标识 |
| **标签 (Tag)** | 附着在被追踪物体上的 UWB 移动节点，实时解算其三维位置，3D 中为蓝绿色发光球体 + 垂直投影线 |
| **尾迹 (Trail)** | 标签位置的历史轨迹，随时间衰减淡出 (默认 2 秒窗口) |

## 视图模式

| 术语 | 定义 |
|------|------|
| **2D 模式** | 俯视投影视图，X 轴向右，Y 轴向上，Z 值以数字形式在信息面板显示 |
| **3D 模式** | 三维透视视图，支持轨道旋转(Orbit)和平移(Pan)操控 |
| **模式切换动画** | 2D→3D：摄像机上视角平滑旋转到 45° 斜视角；3D→2D：从当前视角平滑回到上视角后切换 |

## 交互

| 术语 | 定义 |
|------|------|
| **拖拽 (Drag)** | 在画布上直接用指针移动目标点或基站位置，实时同步到后端 |
| **轨道操控 (Orbit)** | 3D 模式下旋转视角 (鼠标拖拽) 和平移 (右键拖拽) |

## 技术栈

| 术语 | 定义 |
|------|------|
| **Canvas 2D** | HTML5 Canvas 原生 2D 渲染引擎，当前用于 2D 模式 |
| **Three.js r152** | 浏览器端 WebGL 3D 渲染库，通过 CDN 引入，用于 3D 模式 |

## 3D 模式设计决策

| 决策 | 结论 |
|------|------|
| 3D 渲染库 | Three.js r152 (CDN) |
| Canvas 方案 | 双 Canvas 叠加，通过 display 切换 |
| 坐标映射 | 物理 [x,y,z] → Three.js [x, z, y]（Y-up） |
| 初始摄像机 | 45°斜视角，位置 (8,6,8) 看向场地中央 (2,0,2) |
| 模式切换动画 | 2D→3D：上视角平滑旋转到 45°；3D→2D：当前视角平滑回到上视角 |
| 模式切换开关 | Segmented Control，位于底部状态栏最左侧 |
| 地面网格 | 扩展范围，超出 -1~6 以获得视觉呼吸空间 |
| 测距连线 | 默认关闭，可选项（通过开关启用） |
| 垂直投影线 | 标签位置向下引虚线到地面 |
| 3D 拖拽 | 关闭（仅保留 Orbit/Pan） |
| 元素标识 | 基站和目标点使用 Sprite 文字标签 |
| 3D 尾迹 | 3D 空间曲线，跟随标签 X,Y,Z 三维位置，体现高度变化 |
| 3D 资源加载 | 懒加载，用户首次点击 3D 时才加载 Three.js |
| 测距连线开关 | 校准面板底部，默认不勾选 |
